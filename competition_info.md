# 任务介绍

在真实科研与模型研发过程中，实验优化往往不是一次性完成的，而是围绕“提出配置—运行试验—读取反馈—决定下一步”的闭环持续展开。研究者通常无法提前获得完整监督，而只能看到有限的中间反馈，例如验证集指标、训练耗时、资源占用、失败日志等。如何在有限预算内利用这些稀疏反馈持续修正实验策略，是自动化科研与智能体研究中的核心问题之一。

本赛题面向 Agent 长程推理与自动化实验控制，聚焦有限预算、稀疏反馈、禁止并行约束下的自动化实验智能体构建。参赛者需要设计一个能够在多轮交互中持续优化实验决策的 Agent：读取历史试验结果，归纳有效经验，决定下一轮模型、超参数或训练策略，并在预算耗尽前输出尽可能优的预测结果。

为保证评测标准化与可复现性，赛题采用金融场景下的图学习任务作为测试任务，并设置两个具体子任务：

1. 产品分类任务：以图节点分类的形式进行评测；
2. 产品推荐任务：在用户-产品的二部图上的图链接预测问题。

两个子任务都面向稀疏监督场景，即测试节点/用户可利用的邻域或历史信息较少。比赛重点不在于手工调参或单一模型结构本身，而在于 Agent 是否具备以下能力：

* 在多轮实验中维护有效的实验记忆；
* 从稀疏反馈中提炼规律并更新后续策略；
* 在预算约束下进行合理的探索、利用与停止决策；
* 从不同数据集的实验过程中总结可迁移经验用于新的图任务。

为便于理解，可以将本赛题看作一个受控的金融模型研发沙盒。

以新用户金融产品推荐为例，Agent 首先需要根据已有用户行为、产品特征和历史交互结果生成一个初始实验方案；随后根据有限反馈判断当前方案的不足，例如部分新用户预测不稳定、某类产品召回不足、模型训练成本过高，或验证结果与线上反馈不一致；接着，Agent 需要自主决定下一轮实验方向，例如调整特征使用方式、改变模型结构、修改训练策略、重新分配实验预算，或在收益有限时提前停止。经过多轮迭代后，Agent 需要在预算耗尽前选择当前最优方案并生成最终提交结果。

因此，本题是一个“受控机器学习实验环境中的长程决策问题”，也是一个兼具研究性与工程性的综合挑战题。赛题不仅关注最终预测结果的准确性，也关注 Agent 系统在有限预算下如何组织多轮实验、利用反馈信息、形成优化策略并逐步改进预测器。

# 二、任务目标

本赛题围绕“稀疏反馈下的自动化实验控制”展开，参赛者需要面向给定的图学习任务，设计一个能够自主开展多轮实验的智能体系统。该系统需要在资源受限条件下，自动完成：

* 读取历史实验结果
* 分析反馈信号
* 决定下一轮实验配置
* 持续迭代优化
* 输出最终预测结果

本赛题包含两个具体子任务：

## 产品分类任务

参赛者需要根据给定图数据（含产品节点特征与节点间连接关系），对测试节点进行类别预测。

## 产品推荐任务

参赛者需要根据给定用户产品交互序列、用户特征和产品特征，为每个测试用户预测一个按置信度排序的产品列表。

两个子任务都面向典型的稀疏监督场景。

* 在产品分类任务中，需要对连接关系较为稀疏的节点进行分类；
* 在产品推荐任务中，需要对仅具有少量交互历史的新用户进行推荐。

参赛方法不能依赖密集监督信号，而需要在有限反馈、有限预算和多轮实验过程中，逐步归纳有效经验并优化决策策略。

参赛者需要构建一个自动化实验 Agent 系统。对于每个待求解任务与数据集，Agent 需要自主决定尝试的预测器模型结构、超参数配置、训练策略或停止时机，并在有限预算内不断改进结果，最终得到预测器来完成预测任务。

比赛提交阶段需要提交预测结果文件，平台根据预测结果自动评测。进入复审阶段的队伍还需提交完整代码，以证明其方法能够在规定环境中稳定复现。

本赛题鼓励结合以下技术：

* 图学习
* 自动化搜索
* 实验记忆
* 规划与反思
* 多 Agent 编排
* RAG
* Qwen 系列模型 API

考虑到复现阶段对代码体积和运行环境的要求，参赛者应优先设计轻量、稳定、可审计的预测器模型与 Agent 系统，而不是依赖难以复现的本地大模型。

# 三、任务数据

本赛题采用 A/B 榜机制。

* A 榜：初赛测试集，用于模型调试、提交格式校验与阶段筛选；
* B 榜：复赛测试集，用于代码提交阶段自动评测排名。

A 榜、B 榜均包含：

* 1 个产品分类数据集
* 1 个产品推荐数据集

任务形式、接口、评测指标保持一致，仅数据划分不同。

# 五、任务规则及提交说明

## （一）提交文件

### 产品分类任务

文件名：

```text
A1.csv
```

格式：

```csv
test_idx,label
18,3
19,7
```

### 产品推荐任务

文件名：

```text
A2.csv
```

格式：

```csv
uid,prediction
u000001,"i000001,i000002,i000003,..."
```

prediction 需：

* 长度为 10
* 按置信度排序
* 使用英文逗号分隔

---

A 榜数据评测时间：

```text
2026/06/08 10:00 - 2026/07/21 20:00
```

B 榜数据评测时间：

```text
2026/07/22 00:00 - 2026/07/24 17:00
```
